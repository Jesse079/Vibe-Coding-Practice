# 开发日志说明

## 文件规则

- 每天一个文件：`YYYY-MM-DD.md`
- 日期使用开发电脑本地日期。
- 同一天继续开发时，在原文件末尾追加新的“会话记录”。
- 不覆盖或改写之前已经记录的事实。

## 每次收尾必须记录

```md
## 会话：简短标题

### 本次目标

- ...

### 完成事项

- ...

### 修改文件

- `path/to/file`

### 验证结果

- ...

### 发现的问题

- 无；或写明实际问题。

### 下一步待办

- ...
```

## 内容要求

- “完成事项”只能写已真实完成的内容。
- “验证结果”必须说明执行了什么检查及结果，不能只写“正常”。
- 未完成内容放在“下一步待办”。
- 日志不包含密钥、Token、Cookie、摄像头画面、用户外貌或其他敏感数据。
