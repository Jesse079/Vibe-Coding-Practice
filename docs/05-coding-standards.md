# 编码规范

## 基本原则

- 使用 TypeScript 严格模式。
- 优先小函数和明确模块，不创建万能工具类。
- 不使用 `any` 绕过类型问题；确有外部边界时使用 `unknown` 并进行收窄。
- 不保留调试打印、无用注释、死代码或未使用变量。
- 不做与当前里程碑无关的重构。

## 命名

- 类型、接口、类：`PascalCase`
- 函数、变量、实例：`camelCase`
- 常量：语义明确的 `camelCase`；全局固定配置可使用 `UPPER_SNAKE_CASE`
- 文件和目录：小写 `kebab-case`
- 布尔值使用 `is`、`has`、`can`、`should` 等前缀
- 时间变量带单位，例如 `hoverDurationMs`

## 模块边界

- 摄像头模块只管理媒体流和视频状态。
- 手部识别模块只包装 MediaPipe 并输出观察结果。
- 手势模块只从关键点推导手势。
- 状态模块只决定允许的状态转换。
- 绘制模块只管理笔画数据和 Canvas 渲染。
- UI 模块负责 DOM、面板和状态反馈。

模块之间通过类型明确的数据对象通信，避免互相直接修改内部状态。

## 状态与时间

- 状态转换集中处理，不能在多个 UI 回调中直接赋值模式。
- 手势计时使用 `performance.now()` 的单调时间。
- 边沿事件必须区分“刚捏合”和“持续捏合”。
- 页面失焦、丢失手部或视频停止时，必须安全结束进行中的临时交互。

## Canvas

- 处理 `devicePixelRatio`，保证高清屏线条清晰。
- CSS 尺寸和实际像素尺寸分开设置。
- 不把绘图状态只存在 Canvas 位图中；笔画数据应可重放。
- 每次绘制前明确设置线宽、颜色、端点和连接样式。
- 不在每帧无条件重绘不变的 DOM。

## CSS 与界面

- 颜色、间距、层级和动画时长使用 CSS 自定义属性集中管理。
- 支持 `prefers-reduced-motion`，关键状态不能只靠动画表达。
- 交互元素保留键盘和鼠标可用性。
- 不用内联样式拼接动态状态，优先使用 class 或 CSS 变量。

## 错误处理

- 面向用户的错误使用容易理解的中文。
- 控制台错误可带技术上下文，但不得包含敏感信息或摄像头数据。
- 不使用空 `catch`。
- 可恢复错误提供重试入口；不可恢复错误明确说明原因。

## 依赖

- 新依赖必须服务于当前里程碑。
- 添加前检查浏览器原生 API 是否已经足够。
- 锁定文件必须提交。
- 不使用来源不明的模型文件、脚本或 CDN。
