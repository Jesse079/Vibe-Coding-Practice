# 开发需求

## 产品目标

创建一个本地运行的桌面网页。用户站在摄像头前，通过单手手势在全屏绘图区隔空写字或画图，并能调整后续笔画的颜色和粗细。

首版优先验证手势识别、状态切换和绘制体验，不追求完整产品功能。

## 目标环境

- 桌面版 Google Chrome
- 电脑摄像头
- 本地开发服务器
- 单人、单手操作

## 功能需求

### 摄像头与画布

- 页面请求摄像头权限。
- 摄像头画面镜像显示为半透明全屏背景。
- 绘制内容覆盖在摄像头画面上方。
- 页面尺寸变化后，坐标映射和已有笔迹应保持合理。
- 摄像头不可用、被拒绝或中断时给出明确提示。

### 模式

系统同一时间只能处于一种模式：

- `idle`：初始和等待手势的空闲模式。
- `configuring`：选择画笔颜色、粗细并完成配置。
- `drawing`：捏合保持期间连续绘制。

配置模式和绘制模式互斥。

### 配置

- 空闲模式下，五指张开稳定保持 0.6 秒进入配置模式。
- 触发期间显示进度反馈。
- 配置面板提供 5 档粗细。
- 配置面板提供白、红、蓝、绿、黄 5 种颜色。
- 默认画笔为白色，粗细使用中间档。
- 单食指作为配置指针。
- 指针悬停目标约 0.8 秒或在目标上捏合均可确认选择。
- 指向“完成”并悬停约 0.8 秒后保存配置并返回空闲模式。
- 新配置只影响之后创建的笔画，不修改已有笔迹。

### 绘制

- 空闲模式下，拇指与食指捏合进入绘制模式并落笔。
- 保持捏合并移动时连续画线。
- 松开捏合立即抬笔，并返回空闲模式。
- 每笔保存创建时的颜色和粗细。
- 轨迹应尽可能平滑，避免明显锯齿、抖动和长拖尾。

### 清除

- 清除按钮固定在界面上。
- 仅空闲模式可执行清除。
- 单食指悬停按钮进入预激活状态，再捏合确认清空全部笔迹。
- 清除不能在配置或绘制过程中误触发。

### 退出

- 摄像头启用后显示退出按钮。
- 点击退出后清空全部笔迹和诊断内容。
- 停止手部识别并释放摄像头媒体轨道。
- 返回“启用摄像头”页面；保留当前画笔颜色和粗细。

### 状态反馈

- 显示当前模式。
- 显示是否识别到手。
- 对五指触发、悬停选择、捏合和清除预激活提供可见反馈。
- 开发诊断模式可显示关键点、帧率和识别状态。

## 首版不做

- 多手或多人识别
- 手机和平板适配
- 用户账号、后端或云端同步
- 画布自动保存
- PNG 或其他格式导出
- 文字识别、笔迹美化或图形识别
- 录制或保存摄像头画面

刷新页面后画布可以清空。
