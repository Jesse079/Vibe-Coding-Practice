# 交互与视觉规范

## 视觉方向

- 已选定方案 1“沉浸舞台”为正式视觉方向。
- 视觉基准：`docs/assets/immersive-stage-reference.png`。
- 深色沉浸式全屏界面。
- 半透明镜像摄像头画面作为背景。
- 彩色笔迹和手势反馈保持高对比。
- 不使用复杂装饰，不让控制元素遮挡主要绘图区。
- 顶部仅显示轻量模式状态，底部使用横向工具坞。
- 配置面板从底部工具坞上方浮出，关闭后恢复大面积绘图区。
- 参考图用于确定层级、布局和氛围，不照搬其中与正式需求冲突的手势文案。

## 界面层级

从后到前：

1. 镜像摄像头视频
2. 深色半透明遮罩
3. 已有绘图 Canvas
4. 手部关键点诊断层（默认可隐藏）
5. 手势指针、进度和模式反馈
6. 配置面板与清除按钮

## 模式规则

### 空闲模式

允许：

- 五指张开计时进入配置
- 捏合进入绘制
- 单食指操作清除按钮

如果多个条件在同一帧成立，优先处理已经开始计时或预激活的交互，避免状态跳变。

### 配置模式

- 绘制手势无效。
- 单食指移动指针。
- 悬停选项约 0.8 秒或指向选项时捏合，可确认颜色和粗细。
- “完成”只使用悬停确认，避免完成按钮上的捏合在退出后立刻变成绘制。
- 完成后需要等待捏合释放，才允许空闲模式再次进入绘制。

### 绘制模式

- 配置和清除手势无效。
- 捏合保持期间绘制。
- 捏合松开后结束当前笔画并返回空闲。

## 配置面板

- 固定在底部工具坞上方，不跟随手移动。
- 包含当前颜色、当前粗细、5 个颜色选项、5 个粗细选项和完成按钮。
- 悬停时显示环形或条形进度。
- 选中项使用清晰的描边、对勾或尺寸反馈，不能只依赖颜色区分。
- 面板打开时保留上一次已保存配置；未完成前的选择可直接作为候选值显示。

## 清除按钮

- 固定在不容易与主要书写区域重叠的位置。
- 单食指进入按钮区域后显示预激活进度或高亮。
- 达到预激活时间后提示“捏合清除”。
- 只有指针仍在按钮区域且发生新的捏合边沿时才清空。
- 清空后提供短暂视觉反馈，不增加撤销功能。

## 状态反馈文案

文案短且直接，例如：

- 正在加载手势识别
- 请允许使用摄像头
- 未识别到手
- 空闲
- 张开手掌以配置
- 正在配置画笔
- 捏合开始绘制
- 松开完成这一笔
- 指向并捏合以清除

## 可用性

- 所有按钮应保留鼠标可操作能力，便于调试和摄像头不可用时检查界面。
- 提供退出配置的鼠标按钮；手势流程仍以“完成”为主。
- 对识别不稳定给出状态反馈，不把失败表现成按钮无响应。
- 页面首次使用应有简短操作提示，但不遮挡整个画面。
