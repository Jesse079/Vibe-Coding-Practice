# 测试与验收

## 测试原则

- 自动测试负责稳定逻辑：状态机、阈值、计时、坐标和笔画数据。
- 手工测试负责真实摄像头、光照、人体动作和视觉体验。
- 每个里程碑只运行与其风险匹配的检查，但生产构建必须在交付前通过。

## 自动测试重点

### 状态机

- 初始模式为 `idle`。
- `idle` 可进入 `configuring` 或 `drawing`。
- `configuring` 不能直接进入 `drawing`。
- `drawing` 不能直接进入 `configuring`。
- 配置完成和绘制松开均返回 `idle`。
- 非法事件不改变状态。

### 手势

- 五指张开未满 0.6 秒不触发。
- 中途失效会清空计时。
- 捏合进入阈值与松开阈值形成稳定滞回。
- 持续捏合不会重复产生落笔事件。
- 单食指判定不能把普通张手误判为指针。

### 绘制

- 落笔创建新笔画。
- 松开后下一次落笔创建另一笔。
- 每笔保留创建时的画笔配置。
- 清除删除所有笔画。
- 缩放映射不会产生无效坐标。

### 交互

- 配置模式的捏合只确认选择，不开始绘制。
- 完成配置后，未松开的捏合不能立即开始绘制。
- 清除仅在空闲、预激活且出现新捏合边沿时执行。

## 手工验收场景

1. 首次打开并允许摄像头权限。
2. 拒绝权限后看到说明，并可重新尝试。
3. 手进入和离开画面，状态反馈正确。
4. 五指张开不足 0.6 秒不打开配置。
5. 进入配置后选择不同颜色和粗细。
6. 完成配置时不产生意外笔迹。
7. 白色画一笔，改成红色后再画一笔，两笔样式分别保留。
8. 快速和缓慢移动时轨迹连续。
9. 捏合临界抖动时不频繁落笔抬笔。
10. 绘制中丢失手部后能够安全结束。
11. 清除按钮路过不触发，预激活后捏合才清空。
12. 调整浏览器窗口大小后坐标仍对齐。
13. 摄像头流中断后显示可恢复错误。
14. 点击退出后清空画布、关闭摄像头并返回启用页面。

## 性能观察

- 页面操作无明显卡死。
- 指针和笔迹延迟不产生明显“追手”感。
- 记录识别帧率和渲染帧率，但不预设脱离设备实际情况的硬指标。
- 如果主线程长任务持续影响交互，再评估 Worker。
