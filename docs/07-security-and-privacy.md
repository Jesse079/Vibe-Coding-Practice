# 安全与隐私规范

## 摄像头

- 只在用户明确允许后调用 `getUserMedia`。
- 摄像头视频、截图和手部关键点只在浏览器内存中处理。
- 不上传、持久化、缓存或写入日志。
- 页面卸载或停止功能时关闭所有媒体轨道。
- 权限被拒绝时不循环弹出请求，不使用诱导性文案。

## 本地运行

- 开发阶段通过 `localhost` 运行。
- 非本地主机部署时必须使用 HTTPS，确保摄像头 API 可用且安全。
- 首版不开放公网服务、不创建后端接口。

## 模型与依赖

- MediaPipe 依赖来自官方 npm 包。
- 模型文件来源和版本必须记录。
- 优先将模型放在项目静态资源中，避免运行时依赖未知第三方地址。
- 安装依赖后保留锁文件，并检查高风险安全告警。
- 不通过关闭浏览器安全策略解决跨域或权限问题。

## 敏感信息

- 不写入 API Key、Token、密码、Cookie、证书或私钥。
- 不提交 `.env` 或本机隐私文件。
- 日志不得包含摄像头画面、用户外貌描述、完整本机路径中的个人信息或浏览器权限数据。

## UI 安全

- 清除操作必须有预激活和主动捏合确认。
- 摄像头、模型或识别失败时，停止手势产生的破坏性操作。
- 页面不可见或失去焦点时暂停交互，防止后台误操作。

## 依赖和代码变更

- 不引入来源不明脚本。
- 不使用 `eval`、动态拼接脚本或不必要的 HTML 注入。
- 用户可见文本使用安全的 DOM 文本写入方式。
- 安全相关规则发生变化时同步更新本文件和决策记录。
