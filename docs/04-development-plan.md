# 开发执行步骤

## 推进原则

- 一次只完成一个里程碑。
- 每个里程碑必须先通过自己的验收条件。
- 未通过时只修复当前阶段，不提前叠加后续功能。
- 每次任务结束更新当日开发日志。

## 里程碑

### M1 规范初始化

任务：

- 创建 `AGENTS.md`
- 创建 `docs/` 标准文件
- 创建 `dev-logs/` 规则和首日日志

验收：

- 文件结构完整。
- 文档职责清晰、互相引用一致。
- 日志规则可以被后续开发会话直接执行。

### M2 视觉方案确认

任务：

- 根据深色沉浸式方向生成三个独立桌面界面方案。
- 说明各方案的布局和交互重点。
- 等待用户选择或提出组合修改。

验收：

- 用户明确选定最终视觉方向。
- 选定方案覆盖摄像头背景、画布、状态反馈、配置面板和清除入口。

### M3 工程骨架

任务：

- 初始化 Vite + 原生 TypeScript。
- 建立基础目录、样式入口和开发脚本。
- 配置 TypeScript 检查与生产构建。

验收：

- `npm run dev` 可启动。
- `npm run build` 成功。
- 页面无控制台错误。

### M4 摄像头与坐标层

任务：

- 请求摄像头权限。
- 显示半透明全屏镜像视频。
- 实现统一的裁切和坐标映射。
- 实现权限与设备错误提示。

验收：

- 正常、拒绝和不可用状态均可辨识。
- 窗口缩放后视频布局正确。

### M5 单手关键点识别

任务：

- 安装和初始化 MediaPipe Hand Landmarker。
- 只识别一只手。
- 加入可隐藏诊断层。

验收：

- 关键点跟随单手移动。
- 视觉坐标与镜像方向一致。
- 显示识别状态和近似帧率。

### M6 手势与状态机

任务：

- 实现三个模式和事件驱动转换。
- 实现五指张开、单食指、捏合及滞回。
- 实现触发计时和手势释放门控。

验收：

- 合法转换正确。
- 配置与绘制不会串台。
- 临界捏合不会快速反复切换。

### M7 绘制引擎

任务：

- 实现落笔、连续绘制和抬笔。
- 实现轻量滤波和曲线平滑。
- 支持每笔独立颜色与粗细。

验收：

- 多次抬笔不连接。
- 轨迹无明显锯齿或长拖尾。
- 旧笔迹不受新画笔配置影响。

### M8 配置面板

任务：

- 实现 5 种颜色和 5 档粗细。
- 实现食指指针、悬停和捏合选择。
- 实现完成流程和释放门控。

验收：

- 选择反馈清晰。
- 打开和操作面板期间不会产生笔迹。

### M9 清除与防误触

任务：

- 实现空闲模式下的预激活和捏合清除。
- 加入鼠标调试入口和清除反馈。

验收：

- 配置和绘制期间不能清除。
- 仅悬停或路过按钮不会删除作品。

### M10 整体验证与调优

任务：

- 测试低光、快速移动、丢手、临界捏合、窗口变化和视频中断。
- 根据测量结果调整阈值、平滑和识别频率。
- 必要时再评估 Web Worker。
- 更新全部文档和最终日志。

验收：

- 桌面 Chrome 本地完整流程通过。
- 构建、自动测试和手工验收结果有记录。
- 没有已知的高风险误删或模式串台问题。
